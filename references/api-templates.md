## MQ 消息模板

### `[Topic / EventName]`

| 项 | 内容 |
|---|---|
| 业务说明 | [事件表示什么业务事实] |
| Topic/Queue | `[topic]` |
| 生产方 | `[service-name]` |
| 消费方 | `[consumer-a]`, `[consumer-b]` |
| 投递语义 | 至少一次 / 至多一次 / 恰好一次 / [待补充] |
| 幂等要求 | [消费者如何幂等] |

#### 消息字段

| 字段 | 类型 | 必填 | 业务含义 | 兼容性要求 |
|---|---|---|---|---|
| `event_id` | string | 是 | 事件唯一标识 | 不能删除 |
| `occurred_at` | int64/string | 是 | 事件发生时间 | 不能删除 |

#### 消费注意事项

- 消费方必须支持重复消息幂等。
- 新增字段必须兼容老消费者。
- 删除字段或修改字段语义必须走跨服务 RFC。
